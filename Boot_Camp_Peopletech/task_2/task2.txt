Compiling Kernel 6.12 for RPI5 on ubuntu 22.04 PC :-

https://www.linkedin.com/pulse/raspberry-pi-5-kernel-build-guide-david-zhu-pkihc/


Step 1 :- Install build Dependencies :-

$ sudo apt install gcc-aarch64-linux-gnu binutils-aarch64-linux-gnu

$ sudo apt-get update -y && sudo apt-get install -y git bc bison flex libssl-dev

$ sudo apt-get install -y make libc6-dev libncurses5-dev crossbuild-essential-arm64


Step 2 :- Clone / Download Raspberry Pi Kernel Source

Download RPI5 Kernel Version 6.12 :- https://github.com/raspberrypi/linux


Step 3 :- Set Environment Variables

$ export ARCH=arm64

$ export CROSS_COMPILE=aarch64-linux-gnu-


Important: Notice that we do NOT set KERNEL=kernel8 for Pi 5 builds. This is only used for Pi 4 and earlier models.

Step 4 :-Configure and Build

$ make bcm2712_defconfig

$ make -j$(nproc) Image modules dtbs

Flashing kernel Images on Sdcard :-

Note :- The kernel build system doesn't automatically create kernel_2712.img. This is a bootloader naming convention, not a build system feature.


# Copy kernel image with correct name for Pi 5

$ cp arch/arm64/boot/Image /path/to/boot/partition/kernel_2712.img


# Copy device tree files

$ cp arch/arm64/boot/dts/broadcom/*.dtb /path/to/boot/partition/


# Install modules

$ sudo make INSTALL_MOD_PATH=/path/to/root/partition modules_install



For Raspberry Pi 5 Kernel Builds:

DO use ARCH=arm64 and CROSS_COMPILE=aarch64-linux-gnu-

DO use make bcm2712_defconfig for Pi 5-specific configuration

DO manually copy arch/arm64/boot/Image to kernel_2712.img

DON'T use KERNEL=kernel8 (that's for Pi 4)

DON'T expect automatic generation of kernel_2712.img
